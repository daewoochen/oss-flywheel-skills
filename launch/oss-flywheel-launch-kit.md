# OSS Flywheel Skills Launch Kit

Use this file as the first public distribution pack for the repository:

- Repo: [daewoochen/oss-flywheel-skills](https://github.com/daewoochen/oss-flywheel-skills)
- Angle: practical agent skills for turning real open-source work into README upgrades, demos, launches, and contributor momentum

## Core Positioning

### One-line pitch

I made a small collection of Codex-compatible skills for the parts of open source that usually decide whether a repo gets ignored or shared: README clarity, demo proof, launch copy, and contributor onboarding.

### Short pitch

Most skill repos focus on infra or generic prompting. This one focuses on open-source momentum: sharper repo pages, better launch kits, faster demo planning, and smoother contributor ramps.

### Who it is for

- Indie hackers shipping in public
- Open-source maintainers trying to grow a repo
- Small product teams who want better demos and launches

### Why it is different

- The skills are built around distribution loops, not generic "AI productivity"
- Each skill is narrow, practical, and easy to try
- The repository is designed to help projects earn attention after the code exists

## Reply Bank

### If someone asks "Why not just write prompts?"

Because generic prompts are hard to reuse. Skills are easier to install, easier to remember, and better at encoding a repeatable workflow.

### If someone asks "Why this instead of generic AI tooling?"

This repo is narrower on purpose. It focuses on the moments where open-source projects win or lose attention after the code already exists.

### If someone asks "What should you add next?"

Likely candidates:

- benchmark presentation
- docs landing-page structure
- before/after screenshot generation
- release checklist packaging

## What To Avoid

- Do not call it an "AI growth hack" repo
- Do not oversell stars before the repo earns them
- Do not post the same exact wording to every community
- Do not lead with infrastructure terms if the audience cares about outcomes

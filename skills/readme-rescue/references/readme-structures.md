# README Structures

## Fast Activation README

Best for CLI tools, APIs, and small libraries.

1. Headline
2. One-sentence outcome
3. Quickstart
4. Example input and output
5. Key features
6. Limits and docs links

## Trust-Heavy README

Best for developer tools and infrastructure.

1. Headline
2. Why this exists
3. Screenshots, benchmarks, or examples
4. Quickstart
5. How it works
6. Limitations
7. Roadmap or comparison

## Common Quickstart Pattern

- Install
- Run one command
- Show expected output
- Link to deeper setup only after first success

## Useful Questions

- What should the reader do in the first 3 minutes?
- What output should they expect?
- What is the safest supported path?
- What limitation will surprise them if you do not say it early?

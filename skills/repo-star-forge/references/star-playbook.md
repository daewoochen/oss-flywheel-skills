# Repo Star Playbook

## First-Screen Checklist

- Can a stranger tell what the project is in one sentence?
- Is the target user obvious?
- Is there a screenshot, GIF, CLI transcript, or output sample near the top?
- Is there a quick way to try the project in under 3 minutes?
- Does the README show why this exists before explaining every feature?

## Strong README Order

1. Headline with the concrete outcome
2. One short subhead with audience and use case
3. Screenshot, demo, or output sample
4. Fast quickstart
5. Why it is different
6. Key features
7. Realistic limitations
8. Links to docs, examples, or roadmap

## Discoverability Levers

- A clear repository description
- Focused GitHub topics
- Example projects or sample outputs
- Release notes that explain user impact
- A memorable project name and tagline

## Trust Signals That Are Honest

- Benchmarks with methodology
- Compatibility matrixes
- Example repos or public demos
- Limitations and tradeoffs
- Screenshots from real usage

## Avoid

- Fake social proof
- Generic AI buzzwords in the headline
- Huge feature lists above the quickstart
- Walls of badges with no explanation
- Overselling a weak or unfinished product

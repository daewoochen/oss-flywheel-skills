# Demo Blueprint

## The Simple Demo Arc

1. Context: what the user is trying to do
2. Input: the single action that starts the flow
3. Transformation: what changes on screen or in output
4. Payoff: why the result matters

## Shot List Template

- Shot 1: starting state
- Shot 2: triggering action
- Shot 3: moment of change
- Shot 4: final output
- Shot 5: optional proof or comparison

## Good Demo Traits

- Under 60 seconds
- One hero message
- Visible payoff
- Minimal setup
- Easy to narrate in one breath

## Common Failure Modes

- Too many features in one clip
- No clear before and after
- Showing setup instead of value
- Tiny text or crowded screenshots
- Explaining architecture before showing the result

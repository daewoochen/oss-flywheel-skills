# Distribution Kit

## Release Note Order

1. Headline
2. Why this release matters
3. New capabilities
4. Improvements and fixes
5. Breaking changes or migrations
6. Links and next steps

## Good Headline Patterns

- "Faster local runs for ..."
- "New workflow for ..."
- "Less setup for ..."
- "Now with ..."

## Short-Form Post Template

- One line on the new value
- Two or three bullets with concrete wins
- One line on who should try or upgrade

## Keep These Explicit

- Breaking changes
- New defaults
- Migration steps
- Performance claims with context
- Features that are still experimental

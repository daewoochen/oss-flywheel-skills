# Contributor System

## First-Hour Flow

1. Clone the repo
2. Install dependencies
3. Run one known-good command
4. Verify a passing test or visible output
5. Pick a bounded issue

## Good First Issue Traits

- Small surface area
- Clear acceptance criteria
- One main file or subsystem
- Easy local verification
- A maintainer is ready to review

## Maintainer Hygiene

- Label starter-friendly issues clearly
- Keep setup docs current
- Explain how to run checks
- Mention common failure modes
- Respond to first-time contributors quickly
